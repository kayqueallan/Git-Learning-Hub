// O QUE E O GIT? --> Sistema que permite registrar mudancas no codito e matenla facilmente organizadas, como se fosse um historia completa de todas alteracoes que foram mudadas no codigo. Cogido pode ser facilmente conpartilhado. (SISTEMA DE VERSIONAMENTO) //COMMITIS --> Linha de historia / linha do tempo com todas versoes do projeto / modificacoes e manipular essas versoes . Cada vez que e criado uma nova versao e chamado de COMMIT. - versao 1.0 Layout - versao 2.0 Mudanca de todo layot (NEW commit) //BRANCH | RAMIFICACOES --> linha principal e chamada de master, linha do tempo onde seu projeto esta rodando/ toda evolucao. ex: imagine que o projeto tenha 5 evolucoes, mas voce quer adicionar uma nova funcionalidade sem prejudicar o que esta em producao entao e criado uma ramificacao (branch). La pode fazer as alteracoes que quiser e depois fazer um Merge que e juntar a ramificacao ao master. //ABRIR TERMINAL CTRL + ALT + 1 = TEMINAL //COMANDOS --> git init = // INICIALIZAR O REPOSITORIO VAZIO git status = // VERIFICAR COMO ESTA O STATUS / QUANDO ACIONADO MOSTRA SE HA COMMIT E BRANCH/ E MOSTRA SE A PASTA FOI INCORPORADA OU NAO DENTRO DO git git add = // SERVE PARA ADICIONAR O ARQUIVO E ELE SER COMMITADO | ex: git add nome-do-arquivo git commit -m "DETRICAO" = // COMANDO PRINCIPAL PARA ADICIONAR UMA VERSAO DESSE CODIGO, APOS E COLOCADO UMA PEQUENA DESCRICAO QUE FALA BREVE O QUE FEZ NAQUELE COMMIT
